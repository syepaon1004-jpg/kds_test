# 현장 실험의 표본 크기(모수) 설정 — 학습자료

**작성일**: 2026-05-29
**맥락**: KitchenFlow KDS 현장 검증 테스트 설계 시 "몇 건을 관찰해야 신빙성이 있는가?"에 대한 판단 근거 정리

---

## 1. 왜 표본 크기가 중요한가

변수를 통제할 수 없는 현장 실험에서는, 관찰 건수가 적으면 "효과가 있어 보이지만 사실은 우연"이거나 "효과가 있는데 데이터가 부족해서 안 보이는" 상황이 발생한다.

표본 크기는 이 두 가지 실수를 일정 수준 이하로 억제하기 위해 "최소 이 정도는 봐야 한다"를 정해주는 것이다.

---

## 2. Cohen의 효과 크기 (Cohen's d)

### 2.1 개념

Jacob Cohen(1988)이 제안한 "두 집단 간 차이의 크기를 표준화하는 방법."

```
         두 집단의 평균 차이
d  =  ─────────────────────
         데이터의 표준편차
```

예시: 기존 서빙 시간 평균 4분(표준편차 1분), KDS 서빙 시간 평균 3.2분이면

```
d = (4.0 - 3.2) / 1.0 = 0.8
```

### 2.2 기준값의 출처

Cohen이 사회과학 분야의 수백 개 연구에서 실제로 관찰된 효과 크기의 분포를 조사하여 경험적으로 설정한 관례(convention)다. 자연법칙이 아니라 "대략 이 정도면 소/중/대라고 부르자"는 합의.

| 크기 | d 값 | 체감 수준 | 비유 |
|---|---|---|---|
| 소 (small) | 0.2 | 숫자로는 보이지만 눈으로 체감하기 어려움 | "미세하게 다르다" |
| 중 (medium) | 0.5 | 주의 깊게 보면 차이가 보임 | "아, 다르긴 다르네" |
| 대 (large) | 0.8 | 누가 봐도 확실한 차이 | "확 다르다" |

Gignac & Szodorai (2016)의 메타분석에 따르면, 실제 심리학 연구에서 관찰되는 효과 크기는 대부분 0.1~0.3 범위이며, d=0.5 이상은 상당히 큰 효과에 해당한다.

### 2.3 주의점

- d 값은 분야마다 "크다/작다"의 기준이 다르다. 의약품 임상시험에서 d=0.2도 의미 있을 수 있고, 프로세스 구조 변경에서 d=0.8도 기대할 수 있다.
- 핵심은 "이 숫자가 절대적 기준이 아니라 상대적 가이드라인"이라는 점이다.

## 3. 검정력 분석 (Power Analysis)

### 3.1 개념

"실제로 효과가 있을 때, 실험 데이터로 그 효과를 탐지할 수 있는 확률"을 계산하는 방법.

4가지 파라미터가 연결되어 있어서, 3개를 정하면 나머지 1개가 결정된다:

```
효과 크기(d) × 표본 크기(N) × 유의 수준(α) × 검정력(Power)
```

| 파라미터 | 의미 | 관례적 값 |
|---|---|---|
| 효과 크기 (d) | 기존 vs 개선안의 차이 크기 | 0.2 / 0.5 / 0.8 (소/중/대) |
| 표본 크기 (N) | 관찰 건수 | **이걸 구하는 게 목적** |
| 유의 수준 (α) | "효과 없는데 있다고 잘못 말할 확률" 상한 | 0.05 (5%) |
| 검정력 (Power) | "효과 있을 때 탐지할 확률" | 0.8 (80%) |

### 3.2 표본 크기 산출 결과

Power = 0.8, α = 0.05, 독립표본 t검정 기준:

| 기대 효과 | d | 그룹당 N | 총 N |
|---|---|---|---|
| 대 | 0.8 | 26 | 52 |
| 중 | 0.5 | 64 | 128 |
| 소 | 0.2 | 394 | 788 |

### 3.3 동전 비유로 이해하기

동전이 공정한지(앞뒤 50:50) 테스트한다고 하면:

- 동전이 **크게 치우쳐** 있으면 (앞면 80%) → **적게 던져도** 이상함이 보인다 → N 작아도 됨
- 동전이 **살짝 치우쳐** 있으면 (앞면 52%) → **많이 던져야** 우연과 구분된다 → N 커야 함

같은 원리로, KDS가 기존 대비 "확 다르면" 적은 관찰로도 탐지 가능하고, "미세하게 다르면" 많은 관찰이 필요하다.

## 4. 이 프레임워크의 근본적 한계

### 4.1 기대 효과의 순환 문제

검정력 분석의 구조:

```
효과 크기를 먼저 추정한다 → 그에 맞는 N을 산출한다 → 실험한다
```

문제: 첫 단계에서 이미 "내 기대"가 들어간다. d=0.8이라고 기대하고 N=26으로 설계했는데, 실제로 d=0.3이면 그 실험은 효과가 있어도 탐지를 못 한다.

이건 동전 비유로 보면 명확하다:

- 5번 던져서 앞면 4번 → "이 동전 80% 앞면이다!" (d가 커 보인다)
- 95번 더 던져보니 전체 100번 중 앞면 55번 → "아, 55%였구나" (실제 d는 작았다)

처음 5번만 보고 판단하면 효과 크기를 과대추정할 수 있다.

### 4.2 같은 세션 내 관찰의 비독립성

Cohen 프레임워크는 각 관찰이 독립(서로 영향을 주지 않음)이라고 가정한다.

하지만 같은 1시간 안의 100건 주문은 독립적이지 않다:

- 중간에 장비 고장 → 이후 주문 전부 영향
- 워커가 흐름을 잃으면 → 연쇄 지연
- 같은 워커 컨디션, 같은 장비 상태, 같은 러시 강도를 공유

따라서 "같은 날 100건"과 "다른 날 50건+50건"은 통계적 가치가 다르다. 후자가 더 신뢰할 수 있다 — 날짜 간 변동(날 효과)을 상쇄할 수 있기 때문이다.

### 4.3 현장 실험에서의 현실적 접근

학술 논문이 아닌 의사결정을 위한 pilot 실험에서는:

- 통계적 유의성(p < 0.05)보다 **방향성 + 체감**이 핵심 판단 기준이다
- Cohen의 N은 "절대 이만큼"이 아니라 **"대략 이 정도면 우연이 아닐 가능성이 높다"**의 가이드라인이다
- 구조적 변경(프로세스 단계 삭제 등)은 효과가 크게 나타나는 경향이 있어서, 적은 N으로도 방향은 보인다
- 미세 개선(UI 색상 변경 등)은 큰 N이 필요하다
