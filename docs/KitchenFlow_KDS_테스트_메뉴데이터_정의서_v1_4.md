## 2. 면처리 과정 분류 기준

레시피 조리방법에 명시된 후처리 공정으로 분류:

| 코드 | 의미 | 표시 스텝 | 해당 패턴 |
|---|---|---|---|
| torim | 세척+냉각+토렴 | 면세척 → 냉각 → 토렴 | 온면 (뜨거운 육수를 부어 서빙하는 메뉴) |
| cool | 세척+냉각 | 면세척 → 냉각 | 찬면 (차가운 상태로 서빙하는 메뉴) |
| wash | 세척만 | 면세척 | 면 식감을 살리기 위해 세척만 하는 메뉴 |
| none | 후처리 없음 | (없음) | 삶은 후 바로 담는 메뉴 |

**분류 근거**: 각 레시피의 조리방법 텍스트에서 "세척", "냉각", "토렴" 언급 여부로 판단. 추측하지 않음 — 레시피에 명시되지 않은 공정은 none으로 분류.

## 6. 조리시간 선택지 매핑

점장용 카드 입력 화면의 조리시간 6개 박스와 실제 초(sec) 값 매핑:

| 표시 | sec | 해당 면 종류 |
|---|---|---|
| 1:40 | 100 | 생면 1.0 온면 |
| 2:30 | 150 | 생면 1.0 찬면, 생면 1.2 온면, 메밀면 |
| 3:00 | 180 | 쫄비빔면 |
| 1:00 | 60 | 냉면 (해동) |
| 0:40 | 40 | 쌀국수면 |
| 풀어질때까지 | -1 | 우동면 |

**정합성 영향 (v1.4, 2026-06-03)**: 쌀국수면 조리시간 50→40초 변경으로 카드 생성 UI 조리시간 선택지에 **연쇄 영향** 발생.
- 변경 전(v1.3): 활성 메뉴 중 0:50(50초)을 쓰는 메뉴는 쌀국수 2종뿐이었음.
- 변경 후(v1.4): 쌀국수가 40초로 이동 → **0:50 버튼은 어떤 메뉴에도 매핑되지 않음**(dead), 대신 **0:40 버튼 필요**.
- ⚠ **미반영 연관 문서**: KitchenFlow_KDS_테스트프로그램_설계문서 v1.3 및 화면 상세 설계서 v1.1의 조리시간 선택지는 여전히 `1:40/2:30/3:00/1:00/0:50/풀어질때까지`로 "0:50"을 포함. 본 변경에 맞춰 **"0:50" → "0:40"으로 라벨 변경** 필요(또는 0:40 추가). 설계문서·화면설계서 갱신은 별도 작업으로 처리 권장.

> 참고(이전 이력): 설계문서 v1.1에는 잘못된 선택지(1:30/2:10/0:40)가 있었으나 v1.2에서 레시피북 기준(…0:50…)으로 수정 완료. v1.4 시점에는 그 0:50을 다시 0:40으로 바꿔야 하는 상황(메뉴 데이터 변경 기인).

## 7. 면 종류별 색상 매핑

"주문 해석 단계 삭제"를 더 완벽하게 하기 위해, 면 스테이션 워크 카드에서 같은 면 종류를 동일한 배경색+텍스트 색상으로 강조한다. 워커가 텍스트를 읽지 않고도 "뭘 삶을지"를 색상만으로 즉시 인지할 수 있다.

| 면 종류 | bg (배경) | text (텍스트) | 색상 계열 | 사용 위치 |
|---|---|---|---|---|
| 생면 1.0 | `#E8F5E9` | `#2E7D32` | 초록 | "생면1.0 (1:40)" 등 라벨 |
| 생면 1.2 | `#FFF3E0` | `#E65100` | 주황 | "생면1.2 (2:30)" 등 라벨 |
| 메밀면 | `#F3E5F5` | `#7B1FA2` | 보라 | "메밀면 (2:30)" 라벨 |
| 우동면 | `#FFF9C4` | `#F57F17` | 노랑 | "우동면 (풀어질때까지)" 라벨 |
| 쌀국수면 | `#E0F7FA` | `#00838F` | 시안 | "쌀국수면 (0:40)" 라벨 |
| 냉면 | `#FCE4EC` | `#C62828` | 빨강 | "냉면 (1:00)" 라벨 |

색상 값은 `menu_data.json`의 `noodle_type_colors` 필드에 정의되어 있다. 프로그램은 카드 렌더링 시 메뉴의 `noodle_type`을 키로 이 매핑을 조회해 색상을 적용한다.

**렌더링 규칙**:
- WorkCard의 "면 종류 (조리시간)" 라벨(예: "생면1.0 (1:40)")만 색상 강조 영역.
- 배경: `bg` 색상의 둥근 사각형. 텍스트: `text` 색상.
- 면처리 스텝(면세척/냉각/토렴)은 색상 강조 대상 아님 (기본 텍스트 색).
- 카드 헤더(주문번호, 메뉴명)도 색상 강조 대상 아님 (카드 상태 색상 유지).
